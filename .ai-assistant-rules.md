# AI Assistant Rules for tenniszero.org Project

## 🚨 CRITICAL RULES - MUST READ FIRST

**Any AI assistant working on this project MUST follow these rules strictly:**

### 1. CMS Page Design Rule 🔒
- **File**: `src/routes/vibbyai/cms/+page.svelte`
- **Rule**: MUST use iframe embedding for Sveltia CMS
- **Source**: `src="/admin"`
- **NEVER**: Change to standalone page, card layout, or remove iframe

### 2. Data Source Rule 🔒
- **Strategy**: CMS_ONLY (no GitHub API fallback)
- **Source**: All content from `src/content/` directory
- **NEVER**: Re-enable GitHub API or add multiple data sources

### 3. Backend Layout Rule 🔒
- **Structure**: Maintain consistent admin layout
- **Menus**: Keep Growth menu with "NEW" badge
- **NEVER**: Break responsive design or remove core pages

## 📋 Before Making Changes

1. **Read**: [PROJECT_RULES.md](./PROJECT_RULES.md)
2. **Check**: Current implementation matches rules
3. **Verify**: Changes don't violate core principles
4. **Test**: All pages return 200 status codes

## 🔍 Verification Commands

```bash
# Check CMS page has iframe
curl -s http://localhost:5174/vibbyai/cms | grep -i iframe

# Check data source consistency
curl -s http://localhost:5174/api/debug/data-sources

# Test all backend pages
curl -s -o /dev/null -w "%{http_code}" http://localhost:5174/vibbyai
curl -s -o /dev/null -w "%{http_code}" http://localhost:5174/vibbyai/cms
curl -s -o /dev/null -w "%{http_code}" http://localhost:5174/vibbyai/growth
```

## ❌ Common Mistakes to Avoid

1. **Changing CMS to standalone page**
2. **Adding GitHub API fallback**
3. **Breaking iframe embedding**
4. **Using Svelte 4 syntax**
5. **Removing core menu items**

## ✅ Safe Modifications

1. **Style improvements** (colors, spacing)
2. **Performance optimizations**
3. **New feature additions** (following existing patterns)
4. **Bug fixes** (without breaking core design)

## 🆘 If Rules Are Violated

1. **Stop immediately**
2. **Revert changes**
3. **Check PROJECT_RULES.md**
4. **Ask for clarification**

**Remember**: These rules ensure project stability and user experience consistency!
